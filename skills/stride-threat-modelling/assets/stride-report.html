5. Attack Trees

S-01 · API-key replay — Critical

Goal: Charge a merchant fraudulently via the public payments API (S-01)
├── [OR] Obtain a valid merchant API key
│   ├── Harvest from a leaked repo or client-side bundle
│   ├── Sniff an unpinned mobile / web client
│   └── Phish a merchant integrator
└── [AND] Replay the key from an attacker host
    ├── Key is not bound to source IP / mTLS
    └── Key is long-lived (no rotation) and reuse is unalerted
